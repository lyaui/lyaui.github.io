創造全域執行環境 (Execution Context)

在 JavaScript 裡寫程式碼，會產生一個全域執行環境（Execution Context），其中包含：

  1. 創造全域物件(global object)
  2. 指向 window 物件的變數 this

而在全域物件中的變數以及函數會被放進記憶體中，此階段函數尚未被執行，變數尚未被賦值。
全域執行環境會在執行堆（Execution Stack）的最下層，

呼叫執行函數 (Invocation)

Running a function

In javascript, by using parenthesis()

每一次在 JavaScript 呼叫函數，就會創造：

  1. 一個新的執行環境
  2. this（指向 function）

和上面創造全域執行環境的過程差不多，它會有自己的記憶體空間給變數和函數，然後被放進執行堆中。

以上例而言，當遇到最下面 a() 被執行時會呼叫 a 函數，此時一個新的執行環境(a)被創造，被放進執行堆（Execution Stack）中，誰在最上面的就是正在執行的東西。

此時，如果呼叫另一個函數 b，會停止執行 a 內的程式，然後創造另一個執行環境 b，然後將 b 丟到執行堆的最上層。
而此時沒有其他需要執行的函數，於是開始執行 b，當 b 執行結束後就離開執行堆，然後接著執行 a，a 執行完畢離開執行堆，然後回到下面的全域執行環境。

＊程式排列的順序不重要，函數中剩下的程式碼順序也不影響執行先後，執行的順序與函數呼叫、執行推的順序有關。
