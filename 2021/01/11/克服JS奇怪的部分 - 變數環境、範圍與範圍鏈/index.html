變數環境 (variable environment)

Where the variable variables live

And how they relate to each other in memory
描述你 創造 變數的位置，以及它在記憶體中和其它變數的關係。

以上例來說，這裡的每個變數 myVar 都被定義在自己的執行環境（所處的函數中），每當呼叫函數，就會得到這函數自己的執行環境。因此，儘管 myVar 被宣告 3 次（如果未宣告而直接賦值則會成為全域變數），但因為所處不同的環境，它們是不同的、不一樣的，彼此沒有關聯。

而當 a 被執行完畢後，就會回到全域的環境，此時的 myVar 就是全域時所被賦予的 1。

範圍 (Scope)

Where a variable is available in your code

And if it’s truly the same variable, or a new copy

範圍是變數可以被取用的區域。

範圍鏈 (Scope Chain)

「範圍」代表能夠取用指定變數的地方，而「鏈」是外部環境參照的連結。

函數從 詞彙環境(Lexical environments) 不斷向外部環境（同時也是執行堆下方）尋找變數所處的範圍的過程軌跡。

上例中，因為在 b 中未宣告或復直 myVar，所以在 b 的執行環境中找不到 myVar 的值，要參照它的外部環境(Outer Environment)。

那麼，如何決定要參照哪個外部環境呢？就看 b 所處的詞彙環境，也就是物理上程式碼被寫在哪裡，而 b 被寫在 window 中，此時它的物部環境是 window。在 window 中 myVar = 1，因此 console.log(myVar) = 1。

上圖可看出執行環境取得的變數環境不必是正下方的執行環境，本例 a 雖然呼叫 b，但並不是 b 的詞彙環境。
