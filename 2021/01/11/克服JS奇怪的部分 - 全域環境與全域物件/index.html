克服JS奇怪的部分 - 全域環境與全域物件

Disqus:

名稱/值配對與物件

A name/value pair: A name which maps to a unique value.
The name may be defined more than once, but only can have one value in any given context. That value may be more name/value pairs.

一個「名稱/值」的配對，代表一個名稱只會對應到唯一的值，然而一個值卻可以對應到複數的名稱。
例如：路人 A 的名字(name)是 Iris(value)，而 Iris 也可以是路人 BCDE 的名字。

A collection of name value pairs.

而物件，就是名稱/值配對的組合。

全域物件 Global

不論何時跑 JavaScript ， 它都在執行環境（execution context）中執行，而全域執行環境會創造 2 個東西，無論程式碼何時執行，程式碼都被包在執行環境中，而這兩個東西 JavaScript 都為你建立好了。

  1. 全域物件（Global Object）

    Not inside a function.

    如果你的變數和物件不在函數裡，那它們就是全域物件，而我們可以在任何地方取用它們。

  2. 特殊變數 this
    空白的 JS 檔案已存在全域物件和 this，而此時在瀏覽器中的 this 指的就是 window 物件。

結論

JavaScript 程式碼在瀏覽器中被執行時，會建立執行環境(Execution Context)，並在其中建立並包裹住 1.全域物件、2.指向 window 的 this。

因為全域不存在於任何的函數中，已經是在最外層，也就沒有外部環境(Outer Environment)，此時外部環境會為 null。
