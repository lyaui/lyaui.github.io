克服JS奇怪的部分 - 程式執行、JS 與 undefined

Disqus:

執行

前篇文提到的是創造環境，而此處要講的是執行時的情況，在 JavaScript 跑程式碼時：

  • 第 1 階段創造：JS 設定變數和函數到記憶體中。
  • 第 2 階段執行：創造階段已經設定好所有的東西，現在執行已經寫好的程式碼逐行編譯、轉換成電腦懂的東西。

Undefined VS Not defined

簡單而言，創造階段時依 有沒有被宣告（記憶體中是否存在），決定是 undefined 還是 not defined，然後執行階段時將 undefined 替換成賦予的值。

  • Undefined: 被宣告過 已存在於記憶體中，但尚未被賦值。undefined 是 JavaScript 設定的 初始值，會佔據記憶體空間的特殊關鍵字。
  • Not defined: 未被宣告過，記憶體中根本不存在。

＊最好不要手動去設定賦值 undefined (a = undefined)，而是保離系統預設的狀態。
