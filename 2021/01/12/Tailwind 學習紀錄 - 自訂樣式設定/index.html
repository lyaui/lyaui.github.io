Tailwind 學習紀錄 - 自訂樣式設定

Disqus:

Tailwind 豐富的 utility class 讓使用者能夠進行靈活的開發，然而內建的設定未必符合設計需求，因此 Tailwind 提供了 config 檔，讓使用者可以自由新增編輯，進行更自由客製化的設定。

而除了基本的自訂義樣式操作，以下也會介紹一實用的自定義樣式。

Tailwind Config 新增自定義 class 樣式

官方文件

使用 $ npx tailwindcss init --full 產生 config 設定檔，--full即預設的設定，會建立包含所有 class 變數的詳細設定表，可以透過新增或修改裡面的數值進行自訂義設定。

然而實作上不建議直接變更預設值檔案，會建議另外獨立新增檔案來放自己新增的值，這樣才能區分哪些是自己自訂修改的 class，哪些是系統預設的 class。

因此，將本來的 config 檔更名為tailwind-default.config.js來保留預設值（也可以不留，但就作為一個參考），然後再重新執行 $ npx tailwindcss init（不用加- -full），產生一支新的空白 tailwind.config.js 檔案，利用這個檔案來管理新增編輯的 class 樣式。

  • tailwind-default.config.js：參考用的全部 class 設定
  • tailwind.config.js：自定義的 class 設定

打開 tailwind.config.js 新增自定的 class，注意！若直接將設定放在 theme 中會報錯，因為這樣會整個覆蓋掉預設的 tailwind 檔案，因此要放在 extend 中進行擴充，而 extend 會和 tailwind 的預設值檔案進行合併。

設定完後執行之前設定的編譯指令，編譯完成後就可使用新增的 text-mammoth 和 primary, secondary-100, secondary-200 了。

自定義字型

官方文件

使用預設的字型

Tailwind 內建 sans, serif, mono 三種系列字型，而預設的字型是 fontFamily 的 sans 系列，如果想使用其他內建字型，可以使用 class font-serif 或 font-mono。

引入外部字型（以 Google Fonts 為例）

進入 Google Fonts 選擇欲 Embed 的字型後，以範例來說，我希望引用 Fjalla One 和 Noto Sans TC，於是選擇好字型後，產生 @import 的程式碼，然後將它貼到彙整的 css 檔案中。

將字型引入到 css 後，至tailwind.config.js 中新增 fontFamily，設定命名英文字型 Fjalla One 和中文字型 Noto Sans TC，經過編譯後，就能夠使用 font-en(Fjalla One) 與 font-ch(Noto Sans TC) 的自訂字型了。

自定偽類 Hover, Focus, Active

官方文件

在 Tailwind 中，可以透過在 class 前增加前贅詞，例如 hover:bg-yellow-100 來更改 hover 時的背景色，然而並不是所有的行為都有設定反應互動），官方文件這裡 有明確列出每個屬性在哪些互動下會產生反應，可以作為參考。

例如官方預設 fontSize: ['responsive']，也就是說 fontSize 只有在畫面尺寸改變時才會產生變化，而在其他行為，例如 hover 時自字體大小不會有任何變化，如果希望 hover 時也能改變字體大小，就要在 tailwind.config.js 中的 variants 進行設定。

上例中：fontSize 除了原本的 responsive 外，另外新增了 hover。而修改 variants 有兩點必須注意：

  1. 只有在 [] 陣列中的行為才會觸發：因此新增行為時，記得原有的行為也要複製加入，不然會導致預設的行為觸發失效。
  2. variants 中的行為是有順序性的：因此可能會因為設定的順序，導致某些行為即使設定了依然無法觸發，這點必須多加留意。
