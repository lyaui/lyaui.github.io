小專案學習 - React + Firebase 打造專屬相簿

Disqus:

影片中的 Firebase 版本是 7.16.1，如語法運作有問題請確認版本與官方文件！

最近在學習 React.js，雖然基本語法會使用但不是很熟悉，因此希望能透過網路上經驗豐富的前輩的影片來學習，並會對一些小專案寫下紀錄。

因為著重在 Firebase 結合 React 的使用，所以只會紀錄個人認為的重點，一些比較基本的元件渲染或樣式會省略。而專案中會使用許多 Firebase 的語法，只看程式碼可能會不太理解實際的使用方式，所以會在每個 Firebase 的方法附上相應的官方文件連結。

建立 Firebase 專案

在這個相簿專案中，會利用 Firebase Database 與 FireStorage 來作為資料與圖片的儲存，因此在這邊介紹 Firebase 的基礎用法。

首先進入 Firebase 利用右上角「前往主控台」進入新增專案頁面：

  1. 點選「+ 新增專案」新增專案名稱

  2. 選擇是否啟用 GA 分析，因為不使用 GA 功能所以勾選關閉，然後點選建立專案完成建立。
    3 進入 firebase 專案平台，點選頁面中央的 </> 圖示新增網頁應用程式

  3. 在「將 Firebase 新增至您的網頁應用程式」頁面中輸入專案名稱，然後點選「註冊應用程式」

  4. 此時 Firebase 會提供一串程式碼，將紅框的 script 部分複製起來就完成了。

抓取 Cloud FireStore 即時資料

在這個範例中我們想要即時取得圖片集合(images collection)的資料並渲染到畫面上，因此新增一個名為 useFireStore 的 custom hook 來專門獲取 Cloud FireStore 中的圖片網址資料。

要即時取得 Cloud FireStore 的資料可以利用 onSnapshot() 的方法對資料庫的資料進行監聽。每當資料庫有任何變動（例如上傳新的圖片），就會回傳新的資料。

而一般我們在離開元件進行銷毀時，也會移除安裝的監聽事件，因此利用 useEffect return 的 clean function 來執行移除 onSnapshot 事件的監聽，而 Firebase 官方有提供專門的移除事件的方法，請參考下面官方文件。

  • 使用 onSnapshot 抓取即時一整個集合的資料：官方文件：聽取集合中的多個文檔
  • 使用 orderBy 得到排序的資料：官方文件：使用 Cloud Firestore 訂購和限制數據
  • 利用 unsub() 移除 onSnapshot 監聽事件：官方文件：分離聽眾
